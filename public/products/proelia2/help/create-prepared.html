Help - Creating a Prepared Encounter

In order to run an encounter, you have to prepare it first. Proelia separates the notion of prepared and running encounters so that you can prepare an encounter once, and then run as many times as you like, with each instance of the running encounter living on its own. This allows you to create a library of random encounters that you can spring on your group whenever you feel the need.

You create a prepared encounter from the home screen, either by clicking the "create encounter" button in the encounter display area, or by selecting "New Prepared Encounter" from the menu. Note that if you use the button in the home screen, you will get a popover that allows you to specify various elements of the encounter, such as the number of participants and regions. If you select the menu item, the last-used values in the popover will be used.

A new prepared encounter window will appear.

The encounter window is divided up into several sections:

Encounter information
This section is located in the top-left corner of the window, below the title bar. The encounter's name, category, and game system fields are here. In addition, the notes button and "run encounter" button are located on the right side of this section.
Participant list
This section is located along the remainder of the left side of the window. All of the participants and regions in the encounter are in this section. At the bottom of the list are buttons for adding and removing participants and regions to/from the encounter.
Encounter map and controls
This section takes up the rest of the window. The map (or map grid) takes up the majority of the section. Above it are located the "map tools" that allow you to alter elements of the map's display:
"Show"
There are two toggle buttons here that affect the display of tokens on the map:
Borders
Determines if a border is drawn around each token.
Conditions
Determines if conditions for the participant are displayed with the token. Conditions are displayed in a row along the top of the token.
"Map"
Clicking this button will display a popover that allows you to alter the map itself:

The map tools popover contains a number of tools:
Show
There are two toggle buttons here that affect the map:
Map
Toggles the display of the map background image. If no map background image has been set, this button is disabled.
Grid
Toggles the display of horizontal and vertical grid lines over the map. Grid lines can be displayed on the map regardless of whether the map has a background image, or is just a blank area.
Scale/Size
This row may display a different set of controls depending on whether the map is displaying a background image or a blank area:
Background image mode
A slider control appears here that allows you to change the size of the map's background image. By default, when a background image is set, it is displayed at its original size. You can change this by moving the slider knob or using the up/down arrows.
Blank mode
A pair of text entries appear here that allow you to set the size of the map in horizontal and vertical squares.
Grid Lines
When grid lines are drawn on the map, these controls allow you to set the thickness, color, and style of the lines.
Grid Offsets
When grid lines are drawn on the map, these controls allow you to "nudge" the positioning of the first line (and all subsequent lines) from the top and/or left of the map.
Background
Two buttons are here that allow you to choose a background image for the map, or clear the current image.
"Zoom"
This popup menu allows you to set the zoom level of the map. Everything on the map is affected by this setting, allowing you to shrink or enlarge the map display so that you can fit everything on-screen, or zoom in to get a better view of something.

The participant list contains all of the participants and regions in the encounter. Participants are listed first, in whatever order you determine, and regions are displayed at the bottom of the list.

A participant entry looks like this:

On the left is the participant token, as it will appear on the map (except for size/scale).

On the right is the hand of the lich. If you dare, you can grab it to drag the participant to a new position in the list.

The remainder of the participant entry contains a number of things:

Type selector
This button, in the upper-left corner, lets you select what type of participant this is: a player character (), adversary (), or object (). Note that the color of the dotted-line border around the participant's information also serve to indicate the type of the participant: blue for player characters, red for adversaries, green for objects.
Tools menu
The tools menu is accessed by clicking on the "tools" button (). From this menu, you can:
  • set or clear the token image;
  • set the color and marker text;
  • change the size of the token on the map (the number of squares on-a-side the token takes up);
  • reset the position of the token on the map (to the upper-left corner);
  • duplicate the participant (make an exact copy);
  • remove the participant from the encounter.
Info button
Clicking this button (in the upper-right corner) displays a popover where you can record notes about the participant, or link it to an external source, such as iPlay4e, or a DDI Character Builder or Monster Builder file.
Name
Click the text field at the top of the area to set the name of the participant.
Health values
The vertical group of number values ("health", "max", and "temp") are the health of the participant. "Health" portrays the current health value (known as "hit points" in some game systems); "max" portrays the upper limit for the participant's health; "temp" portrays a buffer of health that the participant may use. Note that:
  • Some game systems prevent the participant's health from exceeding the "max" value.
  • Some game systems use the "temp" value when adjusting (up or down) the health value, and might adjust it automatically.
Conditions
The majority of the lower portion display conditions that are applied to this participant. At the far left is a button () to add a condition to this participant (the button only appears when the mouse pointer is in the area).

There is a special feature regarding the naming of adversary type participants: if more than one adversary has the same name, a special "tag" is created in each of the participants' tokens. The tag appears in the lower-right corner along the token's white border. It looks something like this: . This feature is automatic, and cannot be invoked manually. It only applies to adversary types, and doesn't affect player character or object type participants.

A region entry looks like this:

On the left is a small version of the region, as it would appear on the map (except for size/scale).

On the right is the hand of the lich. If you dare, you can grab it to drag the region to a new position in the list. Note that you cannot drag the region amongst the participants in the list: regions are always displayed at the bottom of the list.

The remainder of the region entry contains a number of things:

Tools menu
The tools menu is accessed by clicking on the "tools" button (). From this menu, you can:
  • set the color;
  • reset the position of the region on the map (to the upper-left corner);
  • duplicate the region (make an exact copy);
  • remove the region from the encounter.
Info button
Clicking this button (in the upper-right corner) displays a popover where you can record notes about the region.
Name
Click the text field at the top of the area to set the name of the region.
Dimensions
The vertical group of number values ("width" and "height") represent the size of the region in map squares.
